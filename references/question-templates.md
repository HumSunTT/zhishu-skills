## 互动提问模板

### 章节结束提问

读完一章后，用这些问题检验理解：

#### 理解类问题
```
📘 理解检验：

1. 这一章的核心观点是什么？用你自己的话说说看。

2. 作者为什么提出这个概念？它解决了什么问题？

3. 这一章和前面章节有什么联系？
```

#### 思考类问题
```
🤔 深度思考：

1. 你同意作者的观点吗？有没有不同的想法？

2. 这个概念如何应用到你的实际工作/生活中？

3. 如果你是作者，你会怎么展开这个话题？
```

#### 文学作品专用
```
📖 文学探讨：

1. 这个角色的行为动机是什么？

2. 这段描写营造了什么氛围？作者是如何做到的？

3. 你觉得故事接下来会怎么发展？
```

#### 学术论文专用
```
📄 学术探讨：

1. 这个研究的方法论有什么优缺点？

2. 结论是否能推广到其他场景？有什么局限？

3. 如果要继续这个研究，下一步可以做什么？
```

### 主动check-in

阅读过程中适时check-in：

```
📍 Check-in：

我们刚读完了 {段落/章节}，我想确认一下：

• 到目前为止，有什么不理解的地方吗？

• 有没有想要深入讨论的点？

• 要继续往下读，还是先聊聊这部分？
```

## 理解测试模板

### 测试类型

#### 1. 概念理解题

```markdown
## 📝 概念理解测试

**问题：** 请解释 {概念} 的含义。

---

**参考答案：**
{标准答案}

**评分标准：**
- [ ] 正确定义核心概念
- [ ] 包含关键特征
- [ ] 举例说明（加分项）
```

#### 2. 判断题

```markdown
## ✅/❌ 判断题

**陈述：** {陈述内容}

你的判断：[  ]

---

**答案：** {✓/✗}

**解析：** {为什么对或错}
```

#### 3. 选择题

```markdown
## 🔤 选择题

**问题：** {问题}

A. {选项A}
B. {选项B}
C. {选项C}
D. {选项D}

你的选择：[  ]

---

**正确答案：** {字母}

**解析：** {为什么选这个，其他选项为什么错}
```

#### 4. 简答题

```markdown
## ✍️ 简答题

**问题：** {问题}

（请用自己的话回答，3-5句话）

---

**参考答案：**
{参考答案}

**评分要点：**
- [ ] 要点1
- [ ] 要点2
- [ ] 要点3
```

#### 5. 应用题

```markdown
## 🔧 应用题

**场景：** {具体场景描述}

**问题：** 基于你学到的 {概念/方法}，你会如何处理这个情况？

---

**参考思路：**
1. {步骤1}
2. {步骤2}
3. {步骤3}

**评分标准：**
- [ ] 正确应用概念
- [ ] 逻辑清晰
- [ ] 考虑实际情况
```

#### 6. 对比分析题

```markdown
## ⚖️ 对比分析题

**问题：** 请比较 {A} 和 {B} 的异同。

---

**参考答案：**

| 维度 | A | B |
|------|---|---|
| {维度1} | {A的特点} | {B的特点} |
| {维度2} | {A的特点} | {B的特点} |

**相同点：**
- 

**不同点：**
-
```

## 提问技巧指南

### 好问题的特征
- 开放性：不是简单的yes/no
- 关联性：连接已有知识
- 实践性：能应用到实际
- 挑战性：激发深度思考

### 提问层次
```
Level 1: 记忆 → "这一章讲了什么？"
Level 2: 理解 → "这个概念是什么意思？"
Level 3: 应用 → "这个方法怎么用？"
Level 4: 分析 → "为什么这样做有效？"
Level 5: 评价 → "这个观点对吗？有什么问题？"
Level 6: 创造 → "如果是你，你会怎么设计？"
```

### 根据材料类型调整

| 材料类型 | 重点问题类型 |
|---------|-------------|
| 技术文档 | 应用、操作、故障排查 |
| 文学作品 | 情感、人物、象征 |
| 学术论文 | 方法、论证、局限 |
| 新闻资讯 | 事实、多方观点、影响 |
| 哲学思辨 | 概念、逻辑、延伸 |
